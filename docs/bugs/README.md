| 2026-06-27 | [netplay_aobj_restore_double_unhalfswap_resim](netplay_aobj_restore_double_unhalfswap_resim_2026-06-27.md) | ✅ **RESOLVED (`PORT && SSB64_NETMENU`, default-on, env-revertible; soak-confirmed bit-identical Linux↔Android 2026-06-27)** — After a rollback resim, fighter joints rotate continuously / lock into a spin (originally read as "DK's leg," confirmed to affect **all** characters — a deterministic subset of joints on every fighter). No crash; `anim` hash never gates a mismatch. Distinct root cause from [netplay_aobj_interp_quantize_resim](netplay_aobj_interp_quantize_resim_2026-06-26.md) (that fix was necessary but not sufficient); spin persisted with `SIM_F32_QUANTIZE=0` and was **bit-identical Linux↔Android**, ruling out float nondeterminism. Ground truth via `SSB64_NETPLAY_RESIM_STREAM_DIGEST_DIAG`: the forced un-halfswap inside `syNetRbSnapApplyFighterJointPoseAndAnimFromBlob` mutates the figatree EVENT32 bytes **in place at the same cursor** (DK joint 8 `stream=0xb802419c`→`0xde6665ea`, then idempotent — a native→halfswap **double-swap**); soak1 showed **10/38 joints** corrupted across both fighters (P0 8/14/21/25/26, P1 22/23/24/27/32), identical on Android. Root cause: the figatree heap is **shared and not snapshotted**, so at restore its bytes are already native; the blob restores a **mid-stream cursor** the load-time walker never memoized as a head, so the byte-order heuristic mis-applies a swap. The `AOBJ_DOUBLE_SWAP` detector missed it (reads only `sUnswappedHeads`, which lacks mid-stream cursors). Genuine halfswapped bytes only appear after a (re)load, which already evicts the memo via `port_aobj_register_halfswapped_range` + lazy parser re-normalize — so the stable restore path must not re-derive byte order. Fix: gate that un-halfswap behind `SSB64_NETPLAY_RESIM_RESTORE_UNHALFSWAP` (default OFF = fixed); intro-Appear / modelpart / Appear-presentation passes (which follow a genuine reload) keep normalizing. Audit hook: a restore step whose `NetRbStreamDiag` digest changes at the **same `ev=`** is a byte-order writer; trust load-time normalization, not restore-time re-derivation. **Took four parts** (gating the same double-swap at every site that re-derives byte order from a snapshot-restored cursor): (1) the `syNetRbSnapApplyFighterJointPoseAndAnimFromBlob` apply-step force pass; (2) a native-memo re-pin (`port_aobj_event32_head_mark_unswapped`) after `ftMainRefreshFigatreeVisual` evicts the walker memo on reattach (Follow-up 1); (3) the duplicate force pass in `syNetRbSnapApplyFighterModelPartsFromBlob` (Follow-up 2); and — the **decisive** one — (4) the **per-resim-tick repair pass** (`syNetRbSnapshotPreSimUnhalfswapGameplayResimAnim` + four finalize presentation paths) which evicted the memo and force-walked mid-stream native cursors **every replay tick**, collapsing joints to `aw=AOBJ_ANIM_NULL` (P1 j27 fwd `af=30/aw=1` vs resim `af=0/aw=NULL`); routed those five callers through a gated `syNetRbSnapResimRenormalizeFighterFigatreeStreams` that no-ops unless `SSB64_NETPLAY_RESIM_RESTORE_UNHALFSWAP=1`, leaving the genuine intro-Appear fresh-load pass intact (Follow-up 3). Note "drop the `forget()` and trust the memo" can't work standalone: the upstream register/evict wipes the memo before the per-head walk. |
